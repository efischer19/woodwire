# Plans Directory

This directory contains planning documents, roadmaps, and implementation plans for the `{{PROJECT_NAME}}` project.

## Purpose

The `meta/plans/` directory serves as a workspace for:

- **Implementation Plans**: Step-by-step plans for features or initiatives before work begins
- **Roadmaps**: High-level project direction and milestone planning
- **Research Notes**: Investigation notes and findings that inform future decisions
- **Migration Plans**: Detailed plans for significant refactors or technology changes

## Guidelines

- Plans are **living documents** — update them as understanding evolves during implementation.
- Completed plans should be kept for historical reference but clearly marked as complete.
- Plans that result in significant architectural decisions should produce a corresponding [ADR](../adr/) to formally record the decision.
- Use clear, descriptive filenames (e.g., `plan-authentication-system.md`, `roadmap-v2.md`).
